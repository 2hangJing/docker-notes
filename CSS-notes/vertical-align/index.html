第一个出现底部边距的原因是：
①img默认的基线为自己的底部
②vertical-align默认属性：元素的基线对齐父元素的基线

这里的边距就是 div 的基线到底部的距离

第一个图片出现边距不对称的原因是：
有vertical-align: middle;属性的元素 中部 对齐的是：（ 父元素“基线” + x-height ） / 2

x-height : "x" 字母高度

通过让一个P 标签高度==父元素高度，并且设置 vertical-align: middle; 让父元素基线刚好设置在 height/2 - x-height/2 。这样内部行内元素有 vertical-align: middle; 就会垂直居中

第一个出现错位的原因：
行内块元素默认基线在底部，如果有文本那么基线位置就是文本底部，例：“x” 底部，“p” 上半部分圈的底部

overflow:auto 创建了一个新的块级上下文
